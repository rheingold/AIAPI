🛡️ Security Filters - Context-Aware Access Control

ℹ️ Context-Aware Filtering: Control which operations are allowed based on:
📦 Target Process (calc.exe, notepad.exe, etc.) + 🔧 Helper Tool (KeyWin.exe, BrowserWin.exe) + ⚡ Command ({CLICKNAME}, {SETPROPERTY}) + 🎯 Parameter Pattern (num*Button, closeButton*)
📖 Filter Format & Pattern Syntax

Filter Structure:

[ACTION] [PROCESS] → [HELPER]::[COMMAND]/[PATTERN]

Examples:

  • ALLOW calc.exe → KeyWin.exe::{CLICKNAME}/num*Button
    Allow clicking number buttons only in Calculator
  • DENY * → KeyWin.exe::{CLICKNAME}/closeButton*
    Block closing any window (universal rule)
  • ALLOW notepad.exe → KeyWin.exe::{SETPROPERTY}/Text=*
    Allow text editing only in Notepad
  • DENY * → BrowserWin.exe::navigate/*://malicious.com/*
    Block navigation to malicious domains in any browser

Pattern Wildcards:

  • * - Match any characters (zero or more)
  • ? - Match single character
  • [abc] - Match any character in brackets
  • [a-z] - Match range of characters
  • {regex} - Use full regex pattern
